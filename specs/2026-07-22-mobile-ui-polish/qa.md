# QA：移动端表现优化

## 范围

仅前端静态资源，无接口 / 数据 / 缓存变更。

- `app/css/styles.css`：新增 Mobile polish 层
- `app/index.html` / `app/trip.html` / `app/trip-collections/index.html`：`viewport-fit=cover`

## 建议设备

- iPhone SE / 窄屏（≤375）
- iPhone 常规（390）
- Android 中等屏（360）
- 带刘海 / Home Indicator 机型（安全区）

## 检查清单

1. **首页**
   - 生成按钮通栏可点，触控区域足够
   - 「填入示例」易于点击
   - 聚焦 textarea 时 iOS 不强制放大页面
   - 无横向滚动

2. **行程页**
   - Hero 标题不溢出、首屏不过高
   - 顶部工具条在窄屏显示短文案（「分享」「新建」），桌面仍为完整文案
   - 复制分享后按钮文案恢复，短/长标签结构不丢失
   - Tab 可横向滑动（右侧有淡出提示），sticky 不被刘海遮挡
   - 工具条 / 卡片阴影在窄屏减弱，不挤占空间
   - 目的地内嵌卡片为白底，航班时间与价格对比清晰
   - 航班价格行在窄屏横向排列
   - 费用表可横向滑动查看
   - 花销删除按钮在触控设备上可见
   - 照片墙高度合理，Lightbox 可操作

3. **浮层**
   - 弹窗自底部升起，底部不被 Home Indicator 挡住
   - 编辑抽屉 / 聊天面板底部输入区留出安全区
   - 聊天 FAB 不挡内容，邀请气泡在 FAB 上方且不溢出屏幕
   - 极窄屏（≤400）气泡文案变为「聊聊」

4. **无障碍 / 动效**
   - `prefers-reduced-motion` 下现有减动仍生效
   - 主要按钮无双击缩放延迟感

## 回归

桌面宽度（≥900）视觉与改前一致：通栏生成按钮、底部弹窗、FAB 缩小等仅在 ≤760 生效。
